Multiple SSH identities

Intro

This post shows how to use more than one SSH identity on different git servers in the same hostname (or different hostnames too).

Keygen

First thing you need to generate the ssh keys, to do this, use the following command:

$ ssh-keygen -f ~/.ssh/mykey -C "mykey"

He will generate the key under the name you provided.

SSH config file

After the key generation, you have to create (or update) your ~/.ssh/config file, after creating, open the file and add these lines at bottom.

Host mykey
HostName gihub.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/mykey

Load the key

Now you need to load the key on git server (such as like github.com) and load the key on your computer. To load the key on computer run the following commands:

$ ps -e | grep [s]sh-agent

$ ssh-add -l # list the current keys

$ ssh-add ~/.ssh/mykey #if she's not there

A few things...

...you should know

This is what you need to do with your ssh alias:

Default address With alias
git@github.com:username/reponame.git git@alias:username/reponame.git

Cloning a repository

$ git clone git@youralias:username/reponame.git

Changing a existing repository

Edit the file .git/config on your git repo and udpate the line

url = https://username@github.com/username/reponame.git

To

url = git@mykey:username/reponame.git
